Язык программирования: Java
Требуемая версия JDK: 18 или выше
================================================================================

1. ТРЕБОВАНИЯ К ОКРУЖЕНИЮ
-------------------------
Для сборки и запуска программы необходимо наличие установленного JDK 18 или выше (Java Development Kit).
Проверьте установку, выполнив в командной строке:
    java -version
    javac -version

Если команды не найдены, установите JDK 18 (например, Oracle JDK 18) 
и добавьте путь к бинарной папке (bin) в системную переменную PATH.

3. ИНСТРУКЦИЯ ПО СБОРКУ (КОМПИЛЯЦИИ)
------------------------------------
1. Откройте терминал (Командная строка, PowerShell или Bash).
2. Перейдите в директорию, куда распакован архив с файлом .java:
   cd путь/к/папке/с/файлом
3. Выполните команду компиляции:
   javac ProducerConsumerSimulation.java

   Результат: В той же папке появится скомпилированные файлы ProducerConsumerSimulation.class, 
	ProducerConsumerSimulation$Consumer.class и ProducerConsumerSimulation$Producer.class. 

4. ИНСТРУКЦИЯ ПО ЗАПУСКУ
------------------------
Запуск программы осуществляется через интерпретатор Java с передачей четырех 
обязательных аргументов командной строки.

Синтаксис:
   java ProducerConsumerSimulation <N> <M> <S> <Count>

Где аргументы:
   <N>     — Количество потоков-производителей (целое число > 0).
   <M>     — Количество потоков-потребителей (целое число > 0).
   <S>     — Размер кольцевого буфера (целое число > 0).
   <Count> — Общее количество сообщений для обработки (целое число > 0).

ПРИМЕРЫ ЗАПУСКА:
----------------
Базовый тест (1 производитель, 1 потребитель, буфер 3, 5000 сообщений):
java ProducerConsumerSimulation 1 1 3 5000

5. ВЫХОДНЫЕ ДАННЫЕ И ЛОГИРОВАНИЕ
--------------------------------
Программа выводит в консоль:
- Детальную историю операций (добавление/извлечение сообщений) с указанием 
  номера потока, номера сообщения и позиции в буфере.
- Итоговое время выполнения программы в секундах с точностью до 4 знаков.

6. ОБРАБОТКА ОШИБОК
-------------------
При передаче некорректных аргументов (отрицательные числа, нечисловые значения, 
отсутствие аргументов) программа выводит сообщение об ошибке и завершает работу 
без запуска потоков.
